# 电商AI翻译输入法 — 产品需求文档 (PRD)

**版本**: v1.0  
**日期**: 2026-03-26  
**定位**: 面向跨境电商从业者的桌面端 AI 翻译输入辅助工具

---

## 一、背景与目标

跨境电商从业者在与海外客户沟通时，需要频繁地将中文翻译成各国语言并输入到各类聊天软件（WhatsApp、Shopify后台、速卖通、亚马逊等）。目前市面上没有一款工具能像输入法一样"输入中文、自动翻译并输出到任意对话框"。

**核心目标**：
- 用户在任意应用的输入框中，通过快捷键呼出翻译浮窗
- 输入中文 → AI翻译 → 一键粘贴回原对话框
- 整个流程不超过 5 秒，体验接近系统输入法

---

## 二、用户故事

| 场景 | 描述 |
|------|------|
| 回复客户 | 用户在 WhatsApp 中收到英文询问，按 Alt+T 呼出浮窗，输入中文回复，按 Enter 翻译，按"粘贴"自动回到 WhatsApp 并粘贴英文 |
| 快捷话术 | 用户提前录入"物流延迟致歉"等常用话术，一键插入浮窗，翻译后粘贴 |
| 多语言切换 | 用户切换目标语言为日语，继续与日本买家沟通 |
| 翻译剪贴板 | 用户复制了一段中文，按 Alt+C，剪贴板内容自动翻译并替换 |

## 三、功能需求

### 3.1 核心功能

#### 翻译浮窗 (FloatingWindow)
- 无边框、始终置顶的浮动窗口，可拖拽移动
- **输入区**：多行文本框，Enter 触发翻译，Shift+Enter 换行
- **输出区**：显示翻译结果（只读）
- **语言选择器**：浮窗顶部可快速切换源语言和目标语言
- **状态栏**：显示"已记录目标窗口: [软件名]"
- **按钮**：翻译 | 复制 | 粘贴到聊天框 | 清空 | 设置
- 翻译中显示加载状态，不阻塞 UI
- Escape 键隐藏浮窗

#### 自动粘贴机制
1. 用户在目标窗口中按热键 → 记录目标窗口句柄 (HWND)
2. 翻译完成后 → 复制到剪贴板
3. 隐藏浮窗 → 恢复目标窗口焦点
4. 模拟 Ctrl+V 粘贴

#### 剪贴板翻译 (Alt+C)
- 读取剪贴板中文内容
- 自动翻译
- 翻译结果写回剪贴板
- 气泡通知提示完成

### 3.2 热键管理
- 全局热键，在任意应用前台时均可触发
- **默认热键**：
  - `Alt+T`：呼出/隐藏翻译浮窗
  - `Alt+C`：翻译剪贴板内容
- 用户可在设置界面自定义，提供图形化录制（点击按钮后按键）
- 热键冲突时给出提示

### 3.3 AI 模型配置
- 支持所有 OpenAI API 格式兼容的服务商（OpenAI、DeepSeek、通义千问、Moonshot、本地 Ollama 等）
- **配置项**：
  - API Key
  - Base URL（默认 `https://api.openai.com/v1`）
  - 模型名称（如 `gpt-4o-mini`、`deepseek-chat`）
  - 请求超时时间（默认 30 秒）
- 提供"测试连接"按钮，验证 API 是否可用
- 翻译 Prompt 模板可自定义，支持变量 `{input_lang}`、`{output_lang}`、`{text}`

### 3.4 语言设置
- 支持语言（面向跨境电商常用）：

| 语言 | 代码 |
|------|------|
| 中文 | zh |
| 英语 | en |
| 日语 | ja |
| 韩语 | ko |
| 法语 | fr |
| 德语 | de |
| 西班牙语 | es |
| 葡萄牙语 | pt |
| 俄语 | ru |
| 阿拉伯语 | ar |
| 意大利语 | it |
| 荷兰语 | nl |
| 泰语 | th |
| 越南语 | vi |
| 印尼语 | id |

- 全局默认源/目标语言可在设置中配置
- 浮窗中可临时切换，不影响全局设置

### 3.5 快捷话术库
- 预存常用话术（名称 + 内容），例如"物流延迟致歉"
- 浮窗中通过"话术"按钮弹出列表，点击即插入输入框
- 支持在设置中增删改话术

### 3.6 系统托盘
- 程序启动后最小化到系统托盘，不占用任务栏
- 右键菜单：打开翻译窗口 | 设置 | 退出
- 左键单击：呼出/隐藏翻译浮窗
- 托盘图标使用"译"字样式图标

## 四、非功能需求

| 项目 | 要求 |
|------|------|
| 性能 | 翻译请求不阻塞 UI，使用后台线程 |
| 持久化 | 配置保存到 `config.json`，关闭后保留 |
| 兼容性 | Windows 10/11，Python 3.10+ |
| 安全 | API Key 本地明文存储（后续可加密） |
| 错误处理 | API 失败时在输出区显示错误信息，不崩溃 |

## 七、配置文件结构 (config.json)

```json
{
  "hotkeys": {
    "show_window": "alt+t",
    "translate_clipboard": "alt+c"
  },
  "api": {
    "api_key": "",
    "base_url": "https://api.openai.com/v1",
    "model": "gpt-4o-mini",
    "timeout": 30
  },
  "languages": {
    "input": "中文",
    "output": "英语"
  },
  "prompt_template": "请将以下{input_lang}翻译成{output_lang}，只返回翻译结果，不要解释：\n\n{text}",
  "ui": {
    "window_x": -1,
    "window_y": -1,
    "opacity": 0.97
  },
  "phrases": [
    {"name": "示例：催单话术", "content": "您好，请问您的订单有什么问题需要帮助吗？"}
  ]
}
```
